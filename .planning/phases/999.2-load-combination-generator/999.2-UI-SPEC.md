# Phase 999.2 — UI Design Contract

> Visual and interaction contract for the load-case + combination-generator UI added to the **truss2d** browser tool.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Grounding rule:** this phase extends an existing vanilla-JS canvas tool. EVERY value below is taken from the
> existing `ui/truss2d/style.css` token system and `ui/truss2d/index.html` / `script.js` patterns, or derived from
> the locked CONTEXT decisions (D-01..D-22). Do NOT introduce a framework, component library, or a second design system.
>
> **Known refinement input (not blocking):** the user has Tekla Structural Designer combination-generator screenshots
> to share for exact family naming + wording. Where this spec names families/labels it follows EN 1990 + RESEARCH;
> treat copy strings as overridable on receipt of the TSD screenshots. Functional/visual contract stands regardless.

## Spacing Scale

Use the existing CSS custom properties verbatim. All are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| `--space-1` | 4px | Icon/glyph gaps, inline padding, row gaps inside cards |
| `--space-2` | 8px | Compact element spacing, button padding, table cell padding-y |
| `--space-3` | 12px | Panel padding, floating-card padding, wizard row spacing |
| `--space-4` | 16px | Results-panel padding, section gaps, wizard page padding |
| `--space-5` | 20px | Results-grid column gap, major layout gaps |

Exceptions:
- **No new spacing tokens.** If the wizard needs a wider gutter, compose existing tokens (e.g. `--space-4` + `--space-2`); do not add a 6th token.
- **Interactive target floor = 24px height** (not the 44px mobile rule): this is a desktop mouse/crosshair tool; existing `.tool-btn` is ~28px. The `(i)` info button is 16×16 visual with a 24×24 hit area. Justified: desktop-only, tailnet-internal, parity with the existing dense toolbar.

## Typography

Use existing `--font-size-*` tokens. New UI is data-dense (tables, factor expressions), so it sits at the small end.

| Role | Size | Weight | Line Height | Token / Notes |
|------|------|--------|-------------|---------------|
| Body / table cells / inputs | 12px | 400 regular | 1.4 | `--font-size-sm` — combination terms, case rows, wizard option labels |
| Label / default body | 13px | 400 regular | 1.5 | `--font-size-md` — body default, wizard descriptive text |
| Section heading (`h3`) | 9px | 600 semibold | 1.2 | existing `.panel-section h3` — uppercase, `letter-spacing:0.5px`, `--color-fg-faint` |
| Card / wizard title | 16px | 600 semibold | 1.2 | `--font-size-lg` — wizard page title, results-panel `h2` parity |
| Numeric (forces, factors, kN) | 12px | 400/500 | 1.4 | `--font-mono` JetBrains Mono — factor values, force columns, governing-combo expressions |

**Two load-bearing weights for new UI: 400 regular + 600 semibold.** (Weight 500 Medium already exists for canvas labels and mono emphasis — carried, not introduced by this phase; do not add new weights.)

Factored expressions (e.g. `1.35G + 1.5W + 1.05·Imposed`) render in `--font-mono` so factors align and the `·` reads cleanly.

## Color

Existing 60/30/10 split, light theme (dark values in parentheses where they differ — all already defined as tokens):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-bg` #f0f2f5 (#0D1117) + `--color-canvas-bg` #fefefe (#0D1117) | App background, canvas surface |
| Secondary (30%) | `--color-surface` #ffffff (#161B22) + floating card `rgba(255,255,255,0.82)` blur | Left panel, case/combination tables, floating cards, results panel |
| Accent (10%) | `--color-accent-bright` #3f51b5 (#6b8acc) | See reserved-for list below |
| Destructive | `--color-danger` #c62828 (#ef5350) / `--color-danger-strong` #b71c1c (#ff7373) | See destructive list below |

**Accent (`--color-accent-bright`) reserved for — exhaustive list:**
1. Active segment of the results-view selector (`[per-case] [per-combination] [ENVELOPE]`).
2. Active/selected row outline in the case table and combination table.
3. Wizard primary buttons (`Next ▸`, `Generate → table`) and the active wizard page indicator.
4. The `(i)` info-button glyph and its open-state ring.
5. Table-header accent on new tables (mirrors existing `--color-accent-bright` results header treatment).

Accent is NOT used for: ordinary table text, all interactive elements, every button (most reuse the neutral `.tool-btn` surface). The single loudest CTA (`▶ SOLVE`, spike-primary yellow `#F5C82E`) is **unchanged and remains the only yellow** — the wizard primary uses accent-bright so SOLVE keeps the top of the visual hierarchy.

**Destructive (`--color-danger`) reserved for — exhaustive list:**
1. Delete a combination row (`✕` per row).
2. Delete / clear a load case that contains loads (gated — see Copywriting).
3. "Clear generated combinations" action.
4. Inline validation error text on factor inputs / case fields.

### New semantic colors this phase

Two small additions, each with a light + dark value and each justified:

**(a) Load-case nature palette** (canvas colour-by-nature per CONTEXT discretion; categorical, muted, must not collide with existing canvas semantics — tension blue, compression red, reaction purple, deflected orange). Add as `--canvas-nature-*` tokens:

| Nature | Light | Dark | Rationale |
|--------|-------|------|-----------|
| Self weight (permanent, placeholder) | #78909c | #90a4ae | Neutral slate — dimmed; auto-compute deferred, so it reads "inactive" |
| Dead (permanent, default active) | #455a64 | #b0bec5 | Strong neutral blue-grey — the always-present gravity case |
| Imposed (variable, occupancy) | #b9770e | #e0a93b | Ochre/amber — distinct from the green legacy load arrow |
| Wind (variable, lateral) | #1f8fb5 | #4db6d6 | Teal/cyan — clearly "lateral / environmental", distinct from tension blue |

Default load-arrow colour `--canvas-load` (#2e7d32 green) remains the fallback when colour-by-nature is OFF (back-compat: legacy untyped loads → Dead, rendered in the Dead colour once typed). Colour-by-nature is gated behind a Display checkbox (`Colour loads by case`) plus an optional `Show active case only` toggle.

**(b) Governing-member canvas highlight** — when a combination is selected (reverse index, D-19), the members it governs get a highlight that must stand apart from all eight existing canvas colours. **Reserve spike-primary yellow `#F5C82E` (dark: same)** as a thick semi-transparent halo/outline ring on governed members. This is its only canvas use (it is otherwise the SOLVE-button fill only), so there is no ambiguity. Add as `--canvas-governing` token.

No other new colors. Tension/compression/reaction/deflected/grid/node/support colors are all unchanged.

### C1 — Load-Case panel contract
- Header: section `h3` "Load Cases" + float `↗` button (via `setupCardFloat`).
- Active-case selector (C2): `<select id="activeCaseSelect">`; label "Active case". New loads (`script.js:246`) tag `caseId = activeCaseId`. **Quick-solve path unchanged** — a default **Dead** case always exists, so a user who never opens this panel still solves (D-03/Pitfall 5).
- One row per case: `[nature swatch] [name] [nature ▾] [load count] [✕]`.
  - nature swatch = 10px square in `--canvas-nature-*` for that nature.
  - `name` inline-editable on click.
  - `nature ▾` = `<select>` over the 4 natures (Self weight, Dead, Imposed, Wind); enum is forward-extensible (D-02) — render any extra enum values the engine returns without code change.
  - load count = e.g. "3 loads" in `--color-fg-subtle`.
- "Add case" `.tool-btn` at the bottom.
- Empty-of-loads state copy: see Copywriting.

### C3 — Combination table contract
- Columns: **`☑ on/off` | `name` | `class` | `terms (factors)` | `governs` | `✕`**.
  - `class` = `STR` / `SLS` chip (small badge like `.support-tag`).
  - `terms` = inline-editable factored expression in `--font-mono` (`1.35G + 1.5Q`); editing a factor re-runs superpose+envelope only (no re-solve, D-09/Pitfall 6 — no spinner, no network).
  - `governs` = count badge "governs N" (reverse index, D-19); clicking the row selects it and triggers C8 canvas highlight.
  - `✕` = delete row (danger, inline confirm).
- Header actions: `Generate combinations…` (opens C4) + `Add row` (manual blank combination) + `Clear generated` (danger).
- Manual and generated rows are visually identical and share one data shape `{name, cls, terms:[{caseId,factor}]}` (D-17) — auto-gen is a head-start, not a separate cage.

### C4 — Generator wizard contract (two-page, D-14/D-16)
- **Overlay:** `.wizard-overlay` dim scrim `rgba(0,0,0,0.32)` over `.canvas-area`; `body.card-dragging`-style `pointer-events` discipline so canvas can't be misclicked behind it. The scrim is the one justified deviation from the project's no-modal norm: this is a focused multi-step config task where a stray canvas click would be destructive; a scrim + Esc-to-close is the minimal, aesthetic-consistent way to focus it.
- **Card:** `.wizard-card` reuses `.panel-section.floating` look (blur, `--shadow-lg`, `1.5px --color-border-subtle`, `--radius-md`); width ~420px; centered; `✕` top-right.
- **Page 1 — Parameters:**
  - `Code ▾` (Eurocode (EN 1990) selectable; BS / ASCE / NBR present but **disabled "coming soon"**, D-11).
  - `Country / NA ▾` (UK selectable; others disabled).
  - Family / limit-state checkboxes: **STR-ULS ☑ (default on)**, **SLS-characteristic ☑ (default on)**; **SLS frequent**, **SLS quasi-permanent**, **EQU**, **GEO**, **Accidental / Seismic** all rendered **visible-but-disabled, greyed, labelled "coming soon"** (D-15) — never restructured later, only enabled.
  - Equation set: **`6.10` ☑ (on)**; `6.10a / 6.10b` disabled "coming soon".
  - Every option carries a C5 `(i)` info-popover citing the EN 1990 clause + a one-line "when to use".
  - Footer: `Next ▸` (accent-bright primary).
- **Page 2 — Preview:**
  - Generated rows grouped by family/class (STR group, SLS group).
  - Each row editable: toggle on/off, rename, inline-edit any factor (D-16).
  - Each row shows the human-readable factored expression (`--font-mono`) **plus a plain leading-action tag** ("Imposed leading" / "Wind leading").
  - Degenerate case (only Dead + Imposed) shows the single `1.35·G + 1.5·Q` row (D-12) with an info note rather than an error.
  - Footer: `◂ Back` (neutral `.tool-btn`) + `Generate → table` (accent-bright primary — **this is the phase Primary CTA**).
- Wizard **calls the engine** for generation (D-08); JS holds **no** γ/ψ values. Preview is generate-only (combination definitions + expressions, no solve); the solve happens when the table is run, per RESEARCH Open Question 2.

### C5 — (i) info-popover contract (reusable)
- Trigger: `.info-btn` — 16×16 circle, `(i)` glyph in `--color-accent-bright`, 24×24 hit area.
- Popover: `.info-popover` small floating card (same blur/shadow/border family as `.panel-section.floating`), max-width 260px, anchored below-right of trigger, `--font-size-sm` body.
- Content: bold clause citation (e.g. "EN 1990 §6.4.3.2, Eq 6.10") + plain-language "When to use" sentence.
- Dismiss: outside-click OR `Esc`; only one popover open at a time; keyboard-focusable; `aria-expanded` on trigger. This is the single info pattern reused everywhere (wizard options + table-header term glossary).

### C6 — Results-view selector + C7/C8 traceability contract
- Selector (D-20): segmented `[per-case] [per-combination] [ENVELOPE]` at top of `#resultsPanel`; active segment = accent-bright; default = **ENVELOPE** when combinations exist, else **per-case** (back-compat with today's single-case results).
  - **per-case:** dropdown to pick a case; tension/compression/reaction/displacement tables show that case's raw forces (today's behaviour, scoped to a case).
  - **per-combination:** dropdown to pick a combination; tables show that combination's factored forces.
  - **ENVELOPE:** tables show worst force per member with a **`Governing combination`** column (C7).
- **Forward (member → combo, D-19):** clicking a member result row (or a canvas member) opens a detail strip listing that member's force in *every* combination, governing combination flagged. The governing expression string is exactly what feeds the export (D-21).
- **Reverse (combo → members, D-19):** selecting a combination row in C3 highlights every member it governs on the canvas via C8 (`--canvas-governing` yellow halo) and badges the count.
- Legibility-at-scale requirement (the Tekla pain being solved): with many combinations, the ENVELOPE table stays one-row-per-member (not one-row-per-combination); the per-combination detail is on-demand via the member-detail strip, never a wall of rows. Governing column truncates long expressions with a tooltip-on-hover full string.

## Backward-Compatibility & State Contract (cross-cutting)

These are visual/interaction contracts the executor must preserve (from CONTEXT discretion + RESEARCH):

1. **Quick path untouched (D-03):** a default **Dead** case exists at init and on legacy load; `solve()` works identically whether or not the Load-Case panel was ever opened.
2. **Legacy JSON (discretion):** a loaded model with no `loadCases` creates the default Dead case and assigns all legacy loads to it (mirror the `memberOverrides` default-on-read pattern). No migration dialog; silent + back-compatible.
3. **No re-solve on combination edits (D-09):** editing factors/toggles re-runs superpose+envelope only — **no spinner, no network call**. A spinner appearing on a factor edit is a defect.
4. **Export string (D-21):** ENVELOPE view drives the per-member governing-combo string written into `tension_members[]/compression_members[]`; top-level `load_combination` retained for back-compat; schema bumps to 1.2 (additive). No visible UI change unless ENVELOPE is active.
