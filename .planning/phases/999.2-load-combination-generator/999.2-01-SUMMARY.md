**Solver-agnostic EN 1990 (UK NA) load-combination engine in solver_core/loads — data-driven code pack, 6.10 STR-ULS + SLS-characteristic generation with category-aware imposed psi0, and linear superposition of both member forces and displacements with a provenance-carrying envelope.**

## Accomplishments
- New `loads/` package importable as `pda_analysis_software.loads` — pure numpy/python, no matplotlib, no printing, no visualization import.
- Eurocode UK code pack as the single source of truth for gamma (G_sup=1.35, G_inf=1.0, Q_sup=1.5), Wind psi0=0.6, and imposed psi0 as a category table (A-D/H=0.7 default, E_storage=1.0); xi=0.925 and psi1/psi2 carried forward-compat (unused this phase).
- `generate_eurocode` emits EN 1990 6.10 STR-ULS combinations (each variable action leading, others accompanying at gamma_Q*psi0, degrading to a single 1.35G+1.5Q row for Dead+Imposed) plus SLS-characteristic rows at gamma=1.0, with human-readable expressions ("1.35·Dead + 1.5·Wind + 1.05·Imposed") and plain leading-action tags.
- `superpose` / `superpose_displacement` / `delta_max` / `envelope` — combine cached per-case forces AND displacements by factors without re-solving; envelope returns per-member governing combo plus a combo->members reverse index.
- 7 analytical pytest cases pass; existing `tests/test_truss2d.py` byte-identical and green (no-regression gate held).
