<objective>
Add a strictly additive FastAPI endpoint `POST /solve/truss2d/combinations` that accepts a
geometry/material/support definition plus a list of typed load cases (and optionally a code-pack
generate spec), solves each non-empty case ONCE through the existing unchanged `Truss2DAdapter`
(linear superposition, D-09), then uses the Wave-1 `loads` engine to generate combinations,
superpose member forces AND displacements, and envelope with per-member governing-combination
provenance. Each generated combination carries its δ_max (D-13) so the SLS deflection check has a
combination-driven value; imposed ψ0 is resolved by each case's category.

Purpose: This is the seam the UI calls (D-06/D-08). The UI never re-implements γ/ψ logic — it
posts cases and reads back combinations + envelope + per-combination δ_max. The existing
quick-solve path stays intact.

Output: additive Pydantic models + one route in `api_server/app.py`; a TestClient test file.
</objective>

    return {"solver": result.solver, "UG": ..., "FG": ..., "member_forces": ..., "meta": ...}

Add `@app.post("/solve/truss2d/combinations")` `def solve_truss2d_combinations(req)`:
1. `n_nodes = len(req.nodes)`; `n_members = len(req.members)`; `n_dof = 2*n_nodes`.
2. For each case with non-empty loads: build a flat `forceVector` length `2*n_nodes`
   (index = `load.nodeId*2 + (1 if load.direction.lower()=="y" else 0)`), build a `TrussModel2D`
   (reuse `req.A` as-is — scalar or list), call `engine.solve(model, "truss2d")`, and cache BOTH
   `result.member_forces` (tension-positive) in `per_case_forces[case.id]` AND
   `result.UG` (displacement column vector) in `per_case_ug[case.id]`. Skip empty cases
   (Pitfall 4). Do NOT re-solve for combinations (D-09).
3. Build the combination list: convert `req.combinations` (CombinationDef) into engine
   `Combination` objects; if `req.generate` is provided, map `req.cases` to engine `LoadCase`
   objects (nature via `Nature(case.nature)`, `category=case.category`), call
   `generate_eurocode(cases, get_code_pack(req.generate.get("code","eurocode_uk")),
   req.generate.get("families",["STR","SLS"]))` and extend the list. Each engine Combination
   already carries `expression` + `leading`.
4. `superpose` each combination over `per_case_forces`; `superpose_displacement` each over
   `per_case_ug` and take `delta_max` of the result (the SLS δ_max, D-13);
   `envelope(combinations, per_case_forces, n_members)`.
5. Return the response (RESEARCH §"Response shape", extended additively with displacements + δ_max):
```python
return {
  "per_case": {cid: {"member_forces": f.tolist(),
                     "displacements": per_case_ug[cid].reshape(-1).tolist()}
               for cid, f in per_case_forces.items()},
  "combinations": [
     {"name": c.name, "cls": c.cls, "leading": c.leading, "expression": c.expression,
      "member_forces": superpose(c, per_case_forces, n_members).tolist(),
      "delta_max_m": delta_max(superpose_displacement(c, per_case_ug, n_dof))}
     for c in combos],
  "envelope": {
     "members": [{"member": m.member,
                  "max_tension_N": m.max_tension, "gov_tension": m.gov_tension,
                  "max_compression_N": m.max_compression, "gov_compression": m.gov_compression}
                 for m in env.members],
     "combo_to_members": env.combo_to_members},
  "meta": {"n_nodes": n_nodes, "n_members": n_members, "n_cases_solved": len(per_case_forces)},
}
```
