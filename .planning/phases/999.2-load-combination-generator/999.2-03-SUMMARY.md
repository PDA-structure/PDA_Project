# Phase 999.2 Plan 03: truss2d Load-Case Layer Summary

**Typed load-case layer in the truss2d browser UI: an always-present default Dead case, per-load caseId tagging, a Load-Case panel (active-case selector + add/rename/set-nature/delete + imposed ψ0 category selector), colour-by-nature canvas rendering behind Display toggles, and additive save/load with legacy default-to-Dead — all built without touching the quick-solve path (D-03).**

## Performance

- **Duration:** ~9 min (implementation)
- **Started:** 2026-06-27T17:03:35Z
- **Tasks:** 2 implementation tasks committed; Task 3 is a browser-UAT checkpoint (pending)
- **Files modified:** 3 modified, 1 summary created

## Accomplishments
- **Load-case data model** (`script.js`): `loadCases` (array of `{id,name,nature,active,category}`) + `activeCaseId`, a `makeCaseId()` id generator, and `ensureDefaultCase()` that creates the default Dead case when empty and repairs a dangling `activeCaseId`. Called at startup, inside `resetAll`, and on legacy load.
- **Per-load tagging** (D-01): the load-entry push now stamps `caseId: activeCaseId`; a quick-path user who never opens the panel still tags to the default Dead case.
- **Imposed ψ0 category** (Plan 01 engine): cases carry a `category` field meaningful only for the Imposed nature — defaulted to `A-D/H` when a case becomes Imposed, cleared otherwise. The UI exposes an imposed-only `<select>` offering `A–D/H (offices/floors)` (`A-D/H`) and `E (storage)` (`E_storage`).
- **Load-Case panel** (C1/C2): a new `.panel-section` titled "Load Cases" above Material Properties — active-case selector (`#activeCaseSelect`), one row per case `[swatch][name][nature ▾][category ▾ if imposed][N loads][✕]`, and an "Add case" button. Floats via the existing `setupCardFloat` with no extra wiring.
- **Colour-by-nature canvas** (CONTEXT discretion): `drawLoads()` colours each load arrow by its case nature via `getComputedStyle` on the new `--canvas-nature-*` tokens when `chkColourByNature` is on (falls back to the legacy load green when off); `chkActiveCaseOnly` filters to the active case. Both gated behind Display checkboxes (MEMORY render-toggle rule).
- **Additive save/load**: `saveModel()` writes `canvas.loadCases` + a `caseId` on every saved load; `loadModelFromJson` restores them, or — for legacy files with no `loadCases` — creates the default Dead case and assigns all loads to it (T-9992-09 mitigation). Schema version unchanged (additive keys only).
- **Quick path preserved (D-03):** `solve()` is byte-functionally identical — it still posts ALL loads to `/solve/truss2d`. The case model is additive metadata at this stage.

## Decisions Made
- **`activeCaseId` is the live source of truth**; the per-case `active` flag is synced from it (`syncActiveFlags()`) so the selector and the save/load round-trip agree.
- **Destructive delete never drops loads silently**: a case with loads triggers a delete confirm, then a reassign-to-Dead-vs-delete-the-loads confirm; the last remaining case can never be deleted.
- **`renderCaseTable()` is typeof-guarded in Task 1** so the model-only commit is independently functional before the panel function exists (clean atomic commits).
