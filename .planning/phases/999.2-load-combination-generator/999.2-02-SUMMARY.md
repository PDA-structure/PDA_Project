# Phase 999.2 Plan 02: Additive Combinations Endpoint Summary

**A strictly-additive `POST /solve/truss2d/combinations` route that solves each non-empty load case once through the unchanged Truss2DAdapter (linear superposition, D-09), then uses the Wave-1 loads engine to generate EN 1990 combinations, superpose member forces AND displacements (per-combination delta_max, D-13), and envelope with per-member governing-combination provenance â€” guarded by six structured-422 input-validation classes.**

## Accomplishments
- New additive endpoint orchestrates per-case solves through `engine.solve(model, "truss2d")` (the registry and adapter are untouched), caching both `member_forces` (tension-positive) and `UG` per case so combinations are exact weighted sums with no re-solve (D-09).
- Response carries the full D-20/D-21 surface: `per_case` (member_forces + displacements), `combinations` (member_forces + expression + leading + per-combination `delta_max_m`), and a provenance `envelope` (per-member `gov_tension`/`gov_compression` + `combo_to_members` reverse index).
- Imposed `category` is passed through each case to the engine so generated combinations resolve the correct psi0 (A-D/H=0.7 default, E_storage=1.0).
- Optional `generate` spec auto-builds combinations from the `eurocode_uk` code pack; manual `combinations` and generated ones share one response shape.
- Six structured-422 guards (`shape_mismatch`, `unknown_case`, `non_finite_factor`, `unknown_nature`, `unknown_category`, `no_loaded_cases`) all route through the existing `SolverDiagnosticError` â†’ `@app.exception_handler(RuntimeError)` plumbing â€” no new handler.

## Decisions Made
- **non_finite_factor is sent as a raw `Infinity`-token body.** Standard JSON (and JS `JSON.stringify`) cannot emit `inf`, so the test posts `content=json.dumps(body)` (default `allow_nan=True` emits the `Infinity` token); the server's `json.loads` accepts it, Pydantic coerces to `inf`, and the `math.isfinite` guard fires. This proves the guard is real rather than dead code.
- **unknown_category is guarded before generation** so the engine's `psi0_imposed[cat]` KeyError (flagged for conversion by Plan 01) is surfaced as a clean 422 instead of an unstructured 500.
