# Phase 999.2 Plan 05: truss2d Governing-Combination Traceability + Export Summary

**Surfaces THE differentiator (D-18/D-19) in the truss2d results UI: a [per-case]/[per-combination]/[ENVELOPE] results selector (D-20), an ENVELOPE one-row-per-member view with a Governing-combination column + a member-detail strip (forward map), a canvas reverse-index halo highlighting the members a selected combination governs (D-19/C8), the SLS-characteristic δ_max driving the deflection check (D-13), and the per-member governing-combo string carried into exportAnalysis at schema 1.2 — additive and non-breaking for the SEED-005 marimo reader (D-21).**

- **Tasks:** 2 implementation tasks (committed together — see Deviations); Task 3 is a browser-UAT checkpoint (pending)
- **Files modified:** 3 modified, 1 summary created (500 insertions / 35 deletions)

## Accomplishments

### Task 1 — Results-view selector (C6) + Governing column/detail (C7) + canvas highlight (C8) + SLS δ_max (D-13)
- **index.html:** `#resultsViewSelector` segmented control (per-case / per-combination / ENVELOPE) at the top of `#resultsPanel`, plus two view-scoped `<select>`s (`#resultsCaseSelect`, `#resultsComboSelect`) and a `#resultsViewHint`. A `Governing combination` `<th class="gov-col">` added to both tension and compression tables (shown only in ENVELOPE). A `#memberDetail` strip placed below the results-grid.
- **style.css:** `--canvas-governing: #F5C82E` token in `:root` and `[data-theme="dark"]`; segmented-control styles (`.results-view-selector/.results-view-seg[.active]/.results-view-pick/.results-view-hint`), `.gov-cell` ellipsis-truncation + hover tooltip, `.member-detail` strip styles (head/title/close/governing-row/flag), `.dmax-gov` label, and a `.combo-selected` row outline for the selected combination.
- **script.js — view-aware renderResults:** `effectiveResultsView()` resolves `'auto'` → ENVELOPE when `hasActiveCombos()` else per-case; `populateResultsSelectors()` syncs the active segment + fills/show-hides the case/combo dropdowns; `forcesForView()` returns per-case (`perCaseForces[caseId]`) / per-combination (`superposeJS(combo)`) / single-solve (`res.member_forces`, back-compat); ENVELOPE builds one row per member from the cached `comboEnvelope.perMember` with the Governing column from `govT`/`govC`; empty-state copy "Run combinations to see which one governs each member."
- **script.js — SLS δ_max (D-13):** `superposeDispJS()` + `slsEnvelopeDeltaMax()` take the worst δ over the on SLS combinations (re-derived from cached per-case `displacements`) and label it with the governing SLS combo; single-solve δ_max is preserved unchanged when no SLS combos exist.
- **script.js — forward map:** `showMemberDetail()`/`hideMemberDetail()` list the clicked member's force in every on combination (governing flagged); wired to result-row clicks (`buildForceRow`) and to canvas member clicks in View mode.
- **script.js — reverse index:** `selectComboForCanvas()` sets `selectedComboName` (toggle); combination rows are clickable (ignoring inner inputs/buttons); `drawMembers()` computes the governed-member set and paints the thick semi-transparent `--canvas-governing` halo underneath each governed member.
- **Live re-combine:** `recombine()` now also refreshes the results panel + canvas so a factor edit re-envelopes the ENVELOPE view and the SLS δ_max with no re-solve / no network (D-09).

### Task 2 — Per-member governing-combo export string + schema 1.2 (D-21, non-breaking)
- **script.js — exportAnalysis:** new `governingComboFor(memberIndex, sense, fallback)` returns the member's `gov_tension`/`gov_compression` from the cached envelope, falling back to the legacy top-level string when no combinations exist. Each tension/compression row gains an additive `load_combination` field. The top-level `load_combination` becomes `'ULS/SLS envelope — see per-member'` only when an active envelope exists (legacy string otherwise). `schema_version` bumped `'1.1'` → `'1.2'`. No existing export field removed or renamed.

## Decisions Made
- **δ_max re-derived in JS** (superposeDispJS) rather than read from the engine's `delta_max_m`, so it stays correct under factor edits (D-09). Engine value cached as `_engine_delta_max_m` for reference.
- **ENVELOPE one row per member per sense** (tension/compression) — a member appears in both tables when different combos govern each sense; never one row per combination (legibility-at-scale, the Tekla pain).
- **Halo painted underneath** the member stroke so the T/C colour reads on top; `selectedComboName` toggles on repeat click.
- **Top-level export string** only becomes the envelope summary when an active envelope exists; legacy string preserved verbatim otherwise.
