> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Scope Note

Phase 1 UI work is an extension of the existing `ui/frame2d/` interface — not a new UI. Two new checkboxes are added to the sidebar Display section, and BMD/SFD diagrams are drawn on the existing canvas. No new pages, no new layout regions, no new color palette introductions. All design values below are derived from the existing `style.css` and `index.html` to ensure zero visual regression.

Source: existing `ui/frame2d/style.css` and `ui/frame2d/index.html` (confirmed by direct file read).

## Interaction Contract

### New Controls: BMD/SFD Checkboxes

**Placement:** Inside the existing "Display" section in `ui/frame2d/index.html`, directly below the `chkDeflected` checkbox row. Follow the identical `checkbox-label` HTML pattern.

```html
<label class="checkbox-label">
  <input type="checkbox" id="chkBMD"> Show bending moment diagram
</label>
<label class="checkbox-label">
  <input type="checkbox" id="chkSFD"> Show shear force diagram
</label>
```

**Behavior:**
- Checkboxes are disabled (greyed out) before a solve has been completed. Set `disabled` attribute on page load; remove after first successful solve.
- Checking either box immediately redraws the canvas. Unchecking immediately clears that diagram layer.
- BMD and SFD are independent — both can be shown simultaneously, or either alone, alongside the optional deformed shape.
- If no solve result is present (e.g. after Reset All), checkboxes reset to unchecked and disabled.

**Auto-scale rule (D-12):** The maximum absolute ordinate across all members is scaled to 20% of the shortest member's pixel length. All ordinates use the same scale factor. This prevents any single large-force member from making other members' diagrams invisible.

### Canvas Rendering: BMD/SFD Polygons

**Drawing order (per canvas redraw):**
1. Grid
2. Members (undeformed)
3. BMD polygon (if chkBMD checked) — drawn below members visually via fill
4. SFD polygon (if chkSFD checked)
5. Deformed shape (if chkDeflected checked) — always drawn last, on top
6. Node markers and support symbols
7. Load arrows

**Polygon geometry per member (D-11):**
- Compute member angle: `angle = Math.atan2(yj - yi, xj - xi)` from screen coordinates.
- Perpendicular offset direction: `(−sin(angle), cos(angle))` for screen space.
- Positive moment (sagging) draws on tension face — below for horizontal members — i.e. perpendicular offset is in the direction of the positive normal (D-13).
- Draw start ordinate from node i, end ordinate from node j, close polygon back along the member line.
- For members with non-linear moment variation (UDL case), interpolate a midspan ordinate: `M_mid = (M_i + M_j)/2 + wL²/8` — insert as a third polygon point at the midpoint.

**SFD convention (D-13):** Positive shear draws on the right face when facing from node i to node j.

## Stress Output Display

TRUST-11 adds `member_stresses` to the API response `meta` dict. The existing "Member Actions" results table (`#tableMemberActions`) must add a Stress column.

| Column position | Header | Format | Unit |
|---|---|---|---|
| After M_j column (last) | "σ (MPa)" | 1 decimal place | MPa |

Conversion: `stress_MPa = stress_Pa / 1e6`

Color coding: reuse existing `.tension` (blue) for tensile stress (positive), `.compression` (red) for compressive stress (negative), `.zero-force` (grey) for |σ| < 0.01 MPa.
