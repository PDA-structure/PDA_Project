# Plan 260505-v7c-01 — Results panel as right sidebar

**Goal:** Move the results panel from above the canvas (uzg layout) to a right-side sidebar next to the canvas, growing vertically. User feedback: "when display is open it crowds the screen for a medium-large frame".

## Tasks

### Task 1 — Wrap canvas-area + results-panel in flex-row container (HTML + CSS)

**Files:** `ui/frame2d/index.html`, `ui/frame2d/style.css`

**HTML:** Wrap `<main class="canvas-area">` and `<div class="results-panel">` in a new `<div class="canvas-row">`. Reorder so canvas-area comes first (left), results-panel second (right). Both inside `<div class="workspace">` after the toolbar `</aside>`.

**CSS:**
- New rule `.canvas-row { display: flex; flex-direction: row; flex: 1; min-height: 0; gap: var(--space-2) }`
- Rewrite `.results-panel`: remove `border-top`, `border-bottom`, `max-height: 30vh`, `flex: 0 0 auto`. Add `border-left: 2px solid var(--color-accent)`, `flex: 0 0 320px` (fixed width), `max-height: 100%`, `overflow-y: auto`.
- Update `.results-grid`: change from `flex-wrap: wrap` (horizontal) to `flex-direction: column` (vertical stack of tables).
- Add `.results-table-wrap { overflow-x: auto }` so wide tables (Member Actions has 9 columns) scroll horizontally inside the 320px column instead of breaking the layout.

### Task 2 — Browser UAT (checkpoint:human-verify)

**Action:** User refreshes `http://127.0.0.1:8000/ui/frame2d/index.html` (Cmd+Shift+R). Verifies:
1. Toolbar still at top, all cards still default closed
2. Canvas occupies the LEFT portion of viewport (most of the width)
3. Click SOLVE → results table appears as a 320px column on the RIGHT, next to the canvas
4. Results scrolls vertically within the sidebar (Reactions / Displacements / Summary stack vertically)
5. Wide Member Actions table scrolls horizontally inside its container (not breaking the sidebar)
6. Display card open + medium-large frame: canvas no longer pushed down (results are on the side, not on top)
7. Earlier baselines preserved

## Constraints (binding)

- Two files only — `ui/frame2d/index.html` + `ui/frame2d/style.css`
- No JS changes (results show/hide via `style.display = 'block'` — same element, new DOM location)
- No new design tokens
- No truss2d changes
- All earlier baselines preserved (sq0 contrast, tke density, u2h button sizing, uzg cards-closed)

## Tradeoffs

- **Canvas shrinks horizontally by ~320px when results visible.** On a 13" laptop (~1280px), canvas gets ~960px which is still usable. On a 27" iMac (~2560px), canvas gets ~2240px — minimal impact.
- **Wide tables (Member Actions, 9 cols) require horizontal scroll inside the sidebar.** Acceptable tradeoff; alternative is making the sidebar wider (400-500px) which eats more canvas. 320px is a reasonable starting point.
