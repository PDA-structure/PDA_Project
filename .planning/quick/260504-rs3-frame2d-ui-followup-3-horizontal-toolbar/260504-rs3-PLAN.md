### Target rules after rs3

```css
/* ── Workspace (rs3 — column flow: toolbar on top, canvas below) */
.workspace { display: flex; flex: 1; flex-direction: column; gap: 0; min-height: 0; }

/* ── Top toolbar panel (rs3 — horizontal wrap-row) ───────────── */
.panel {
  width: 100%;
  min-width: 0;
  background: var(--color-bg);
  border-bottom: 1px solid var(--color-border-subtle);
  padding: var(--space-3) var(--space-3);
  display: flex;
  flex-direction: row;
  flex-wrap: wrap;
  align-items: flex-start;
  gap: var(--space-2);
  overflow-x: auto;
  box-shadow: var(--shadow-sm);
}

/* Cards size to content with a sane minimum, instead of stretching */
.card,
.panel-section {
  flex: 0 1 auto;
  min-width: 180px;
  margin-bottom: 0;          /* gap on .panel handles spacing */
}
```

Notes on the target:

- `.workspace { min-height: 0 }` is added to prevent flex children from forcing the workspace taller than the viewport (a common gotcha when flipping flex-direction).
- `.panel`: width 100%, min-width 0 (overrides previous min-width: 210px), `border-bottom` instead of `border-right`. Padding tightened from `var(--space-4) var(--space-3)` to `var(--space-3) var(--space-3)` to keep the toolbar compact vertically. `gap: var(--space-2)` spaces the cards horizontally; `align-items: flex-start` stops short cards from stretching to match tall ones (e.g. Section Calculator has more inputs than Geometry).
- `.panel-section, .card` shared rule for horizontal-layout sizing. `flex: 0 1 auto` = don't grow, may shrink, base size = content; `min-width: 180px` = force a sane card width even when content is narrow (e.g. Reset View button card would otherwise be ~80px).
- `margin-bottom: 0` on the cards because the `.panel` `gap: var(--space-2)` now provides the spacing — without this override, cards would have a 12px bottom margin AND an 8px gap stacking. Existing `margin-bottom: var(--space-3)` on `.card` and `.panel-section` is overridden.
- `overflow-x: auto`: if the wrap layout still overflows horizontally (e.g. one card has a long label), allow horizontal scroll on the toolbar rather than clipping. Wrap is the primary mechanism; scroll is the fallback.

**Step 1 — Modify the `.workspace` rule (currently line 139).**

Change FROM:
```css
.workspace { display: flex; flex: 1; gap: 0; }
```

Two additions: `flex-direction: column` flips the axis; `min-height: 0` is a defensive add that prevents flex children from forcing the workspace to grow taller than the viewport (a common flex-column gotcha — without it, scrollable content inside a flex item can override the parent's intrinsic height calculation in some browsers).
