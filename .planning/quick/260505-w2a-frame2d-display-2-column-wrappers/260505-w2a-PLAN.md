# Plan 260505-w2a-01 — Display 2-column via explicit wrappers

**Goal:** Fix the 2-column Display layout so "Show diagram values" appears on the SAME ROW as "Show supports" (column 2 row 1), not next to "Show shear force diagram" (column 2 row 5). Replaces the CSS Grid auto-placement approach (vhi) with two explicit `<div class="display-col">` wrappers — deterministic, no auto-placement cursor surprises.

## Root cause (vhi grid auto-placement bug)

The `vhi` approach used CSS Grid with `grid-column: 2` declared on `chkDiagLabels` and following items, expecting them to backfill column 2 starting at row 1. But the CSS Grid auto-placement cursor advances after each placement: after items 1-5 are placed at rows 1-5 col 1, the cursor sits at row 5 col 2, so item 6 (chkDiagLabels with `grid-column: 2`) lands at row 5 col 2 — visually adjacent to chkSFD.

`grid-auto-flow: dense` doesn't reliably backfill items with explicit column placement.

## Fix

Use two explicit `<div class="display-col">` wrappers inside `.display-body`. Each col is an independent flex-column; the parent `.display-body` is flex-row. Layout becomes deterministic — first item of each col aligned at the top, no auto-placement cursor.
