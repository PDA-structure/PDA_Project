## UAT-driven iterations

Visual UAT after the three planned commits surfaced refinements addressed in the same session:

4. **Member label decongestion** — `a7e8d70`
   - Original `drawMemberLabel` always pushed labels to the same perpendicular side, clustering top-chord and diagonal labels at converging nodes on dense trusses
   - Fix: pick the perpendicular sign that points AWAY from the structure centroid (top chord ↑, bottom chord ↓, diagonals fan outward) + bump offset 14 → 18 px + white halo for residual overlap
5. **Apex-at-node force arrows** — `443872e`
   - User flagged that reaction arrows reading "node → outward" looked physically inverted for a force pushed *into* the node by a support
   - Refactor: shared `drawForceArrow(node, axis, value, color, label)` helper used by both `drawLoads()` and `drawReactions()`; head triangle apex now anchors AT the node, shaft extends OPPOSITE to force direction, label sits at the tail outside the structure. Carries the white halo across.
6. **Apex-gap for coincident X+Y arrows** — `dbd9fc9`
   - Pinned-support reaction with both X and Y components had both triangle apexes landing on the exact node pixel and merging into a diamond shape. Pull each apex back by `2 * sc` along its force direction so the two triangles separate while each still visually touches the node.
7. **`chkReactions` visibility toggle** — `d3d8bca`
   - Added "Show reactions" checkbox between Show loads and Node labels in the Display panel; gates `drawReactions()` and drops the legend's "Reaction" row when unchecked so the legend stops advertising a colour that isn't drawn.
   - Note: D-3 ruled out coupling reactions to `chkSupports` (geometry vs output split). This is a separate dedicated control the user requested at UAT — does NOT conflict with D-3.
8. **Label font 10 → 9 px** — `89a6642`
   - Member labels and arrow labels dropped one notch; legend stays at 11 px to maintain header-vs-caption hierarchy. Still scales via the existing `Symbol size ×` input.
9. **Member label offset 18 → 14 px** — `493ef75`
   - Now that labels are 9 px and the centroid-outward flip handles congestion structurally, the bumped offset read as labels floating too far from members. Back to the original 14 px.
10. **Grid tiles full visible canvas** — `e3bc21a`
    - `drawGrid()` only covered `0..canvas.width` in canvas-pixel space. With the view transform applied, any pan/zoom (including Revit-imported models that land at negative coordinates) pushed the visible region outside that range and the grid disappeared. Fix: compute visible window in transformed coordinates and tile from `Math.floor(x0/GRID)*GRID` outward. Default view (`scale=1, tx=0, ty=0`) math reduces to the original range so no regression on normal models.
