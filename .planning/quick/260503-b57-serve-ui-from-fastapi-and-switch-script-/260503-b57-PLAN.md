FastAPI StaticFiles syntax (current FastAPI 0.135.0 / starlette 0.52.1):
```python
from fastapi.staticfiles import StaticFiles
from pathlib import Path

app.mount(
    "/ui",
    StaticFiles(directory=str(Path(__file__).resolve().parent.parent / "ui")),
    name="ui",
)
```

Note: bare `app.mount("/ui", StaticFiles(directory=...))` serves files at
`/ui/<sub>/<file>` but does NOT auto-serve `/ui/<sub>/` as `index.html` — the
user accesses the full path `/ui/truss2d/index.html`. That's exactly what the
task scope specifies, so do NOT add `html=True` (it would also intercept
unknown paths under `/ui` with a 404 page from StaticFiles, which we don't need).

- Replace with: `const API_URL = ''; // relative — UI is served from the same FastAPI process`

Why empty string (not e.g. "/" or `window.location.origin`):
- Empty string + template literal `${API_URL}/solve/truss2d` yields `/solve/truss2d`, which fetch() resolves against `document.baseURI` → same origin. This works regardless of whether the page is served from Tailscale, Render, or localhost.
- `window.location.origin` would also work but is more brittle (e.g. if the UI is ever embedded in an iframe with a different parent origin).
- Leading `/` ensures the path is origin-relative, not path-relative — important because the page is served from `/ui/truss2d/index.html`; a path-relative `solve/...` would resolve to `/ui/truss2d/solve/...` and 404.
