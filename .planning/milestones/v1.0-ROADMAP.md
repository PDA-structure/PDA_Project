# Milestone v1.0: 2D Solver Foundation

**Status:** ✅ SHIPPED 2026-04-18
**Phases:** 1–2
**Total Plans:** 6

## Overview

Established the trusted, production-ready 2D structural analysis platform. Phase 1 hardened the existing solvers (error handling, test coverage, BMD/SFD rendering). Phase 2 evolved the model (per-member properties, section calculator, result export, UI debugging tools). The platform now provides a clean, reliable analysis pipeline from browser UI → API → solver → results.

## Phases

### Phase 1: Trust and Production Hardening

**Goal**: The existing solvers are reliable enough for structural engineers to trust — errors are handled gracefully, the codebase has no known violations, and analytical correctness is verified across representative structural cases
**Depends on**: Nothing (first phase)
**Plans**: 3 plans

Plans:
- [x] 01-01-PLAN.md — Error handling, print removal, and member stress output
- [x] 01-02-PLAN.md — Test suite expansion with 5 analytical cases and equilibrium assertions
- [x] 01-03-PLAN.md — BMD/SFD canvas rendering and stress display in UI

**Requirements satisfied:** TRUST-01, TRUST-02, TRUST-03, TRUST-04, TRUST-05, TRUST-06, TRUST-07, TRUST-08, TRUST-09, TRUST-10, TRUST-11

**Key deliverables:**
- HTTP 422 on unstable structures (was HTTP 500)
- All print() / summarize_results() removed from solver_core

- Global equilibrium assertions (np.isclose) on all tests
- BMD/SFD rendered on frame2d canvas via cubic Hermite interpolation
- Member stress (σ = F/A) displayed in results table

## Milestone Summary

**Key Decisions:**

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| HTTP 422 at exception handler level (not adapter) | Single catch point; adapters don't need try/except | ✓ Good |
| Equilibrium checks target reaction DOFs only (not sum(FG)) | FG = Kp @ UG always sums to 0; reaction-specific check is meaningful | ✓ Good |
| Per-member E/I/A as Union[float, List[float]] with scalar broadcast | Backward-compatible; no API versioning needed | ✓ Good |
| Section calculator in sidebar (permanent, live update) | Faster workflow than modal dialog | ✓ Good |
| Per-member indicator as thick blue stroke on canvas | Visually clear, no additional UI chrome | ✓ Good |
| Blob URL + revokeObjectURL for JSON export | Memory safe; no server-side file storage needed | ✓ Good |
| Horizontal UDL via direction-cosine decomposition for inclined members | Correct mechanics; handles arbitrary member angles | ✓ Good |
| Zoom/pan in truss2d via CSS transform on canvas element | Simple, no coordinate remapping needed | ✓ Good |

**Issues Resolved:**
- HTTP 500 on under-restrained structures replaced with graceful 422
- truss2d UI files were untracked in git — committed during Phase 2 Plan 3
- Hermite cubic underestimates UDL midspan deflection by ~20% — quartic correction applied

**Issues Deferred:**
- Symbol size scaler for truss2d (skipped in Phase 2 UAT; fixed in quick task 260414-roe)
- Further UI/UX polish deferred intentionally — revisit after interchange format (Phase 3) stabilises data model

**Technical Debt Incurred:**
- None significant — architecture held cleanly through both phases
