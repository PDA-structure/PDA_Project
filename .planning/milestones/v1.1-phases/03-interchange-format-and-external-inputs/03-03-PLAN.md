**Helper function `read_sheet_as_dicts(ws)`:**
```python
def read_sheet_as_dicts(ws):
    """Return list of row dicts keyed by header row values, skipping blank rows."""
    headers = [cell.value for cell in ws[1]]
    return [
        {headers[i]: row[i].value for i in range(len(headers))}
        for row in ws.iter_rows(min_row=2)
        if any(cell.value is not None for cell in row)
    ]
```

**Main convert function `convert(xlsx_path, column_map=None) -> dict`:**

1. Open workbook: `openpyxl.load_workbook(xlsx_path, read_only=True, data_only=True)`
2. Read nodes sheet: `read_sheet_as_dicts(wb[cmap["nodes_sheet"]])`
3. Build node ID remapping: `tsd_id_to_index = {}` — map each TSD node ID to a 0-based contiguous index. This is critical because TSD node IDs are database keys with potential gaps (Pitfall 3).
4. Build `nodes` array: `[[float(row[cmap["node_x"]]), float(row[cmap["node_y"]])], ...]`
5. Read members sheet: `read_sheet_as_dicts(wb[cmap["members_sheet"]])`
6. Build `members` array using remapped indices + 1 (1-based for PDA API): `[[tsd_id_to_index[start]+1, tsd_id_to_index[end]+1], ...]`
7. Extract E, I, A per member if available. If E column header contains "kN/m2", convert to Pa by multiplying by 1000. If all members have the same value, use scalar; otherwise use list.
8. Build default fields:
   - `ENForces`: `[[0, 0]] * n_members`
   - `ENMoments`: `[[0, 0]] * n_members`
   - `forceVector`: `[0] * (n_nodes * 3)` (frame2d default — 3 DOF per node)
   - `restrainedDoF`: `[]` (TSD may not export supports — user adds manually in UI)
   - `bars`, `beamPinLeft`, `beamPinRight`, `pinDoF`, `springDoF`, `springStiffness`: all `[]`
   - `udl_x`: `[0] * n_members`
9. Build canvas section with empty/null values (no UI state from TSD):
   ```python
   "canvas": {
       "origin": None,
       "nodes": [],
       "members": [],
       "supports": [],
       "nodeLoads": []
   }
   ```
10. Return dict with `schema_version: "1.0"`, `solver: "frame2d"`, all payload fields, canvas section.

**CLI entry point:**
```python
def main():
    parser = argparse.ArgumentParser(description="Convert TSD Excel export to PDA JSON")
    parser.add_argument("xlsx_path", help="Path to TSD .xlsx export file")
    parser.add_argument("--output", "-o", help="Output JSON file path (default: stdout)")
    args = parser.parse_args()

    result = convert(args.xlsx_path)

    if args.output:
        with open(args.output, "w") as f:
            json.dump(result, f, indent=2)
        print(f"Written to {args.output}", file=sys.stderr)
    else:
        print(json.dumps(result, indent=2))

if __name__ == "__main__":
    main()
```

**Create `pyrevit_exporters/export_to_pda.py`:**

This script runs INSIDE Revit via the PyRevit extension. It cannot be run or tested in the development environment (no Revit available). It must be valid Python that compiles with `python3 -m py_compile`.

The script must be compatible with BOTH IronPython 2.7 and CPython 3.x. Use only `json`, `os`, `sys` from stdlib. No f-strings (IronPython 2.7 does not support them). Use `.format()` instead.

```python
"""
PDA Analysis Software - Revit Analytical Model Exporter

Exports the Revit analytical model (beams, columns, braces) to PDA's canonical
JSON interchange format (schema_version 1.0, solver "frame2d").

Usage: Run this script from a pyRevit button/script in Revit 2023+.

Requirements:
  - Revit 2023 or later (uses AnalyticalMember API, not the deprecated GetAnalyticalModel)
  - pyRevit 4.8+ (for CPython support) or pyRevit with IronPython 2.7.4+

Notes:
  - Revit internal coordinates are in FEET. This script converts to metres (* 0.3048).
  - The exported JSON contains geometry only (nodes, members, default E/I/A).
    Supports, loads, and member property overrides must be added in the PDA browser UI
    or edited in the JSON file before solving.
  - Non-planar 3D models: only X and Y coordinates are exported. Z is ignored.
    Verify your model is approximately planar before using with the 2D frame solver.
"""

import json

# Revit API imports — these only work inside the Revit runtime
from Autodesk.Revit.DB import FilteredElementCollector, AnalyticalMember
from pyrevit import forms, script

FEET_TO_METRES = 0.3048

doc = __revit__.ActiveUIDocument.Document


def xyz_to_metres(xyz):
    """Convert a Revit XYZ point (in feet) to [x, y] in metres."""
    return [round(xyz.X * FEET_TO_METRES, 4), round(xyz.Y * FEET_TO_METRES, 4)]


def get_or_add_node(xyz, nodes_list, tol=1e-3):
    """Return 0-based index of existing node within tolerance, or append new node."""
    pt = xyz_to_metres(xyz)
    for i, n in enumerate(nodes_list):
        if abs(n[0] - pt[0]) < tol and abs(n[1] - pt[1]) < tol:
            return i
    nodes_list.append(pt)
    return len(nodes_list) - 1


# Collect all analytical members (beams, columns, braces) from the active document
# NOTE: This uses the Revit 2023+ API. The old element.GetAnalyticalModel() was
# removed in Revit 2023 and will raise AttributeError.
members_col = FilteredElementCollector(doc).OfClass(AnalyticalMember).ToElements()

if not members_col:
    forms.alert("No analytical members found in the active document.\n\n"
                "Make sure the Analytical Model is enabled and members have "
                "analytical representations.", title="PDA Export")
else:
    nodes = []
    members = []

    for mbr in members_col:
        curve = mbr.GetCurve()
        si = get_or_add_node(curve.GetEndPoint(0), nodes)
        ei = get_or_add_node(curve.GetEndPoint(1), nodes)
        members.append([si + 1, ei + 1])  # 1-based for PDA API

    n_nodes = len(nodes)
    n_members = len(members)

    output = {
        "schema_version": "1.0",
        "solver": "frame2d",
        "nodes": nodes,
        "members": members,
        "ENForces": [[0, 0]] * n_members,
        "ENMoments": [[0, 0]] * n_members,
        "forceVector": [0] * (n_nodes * 3),
        "E": 200e9,
        "I": 1e-4,
        "A": 0.01,
        "bars": [],
        "beamPinLeft": [],
        "beamPinRight": [],
        "restrainedDoF": [],
        "pinDoF": [],
        "springDoF": [],
        "springStiffness": [],
        "udl_x": [0] * n_members,
        "canvas": {
            "origin": None,
            "nodes": [],
            "members": [],
            "supports": [],
            "nodeLoads": []
        }
    }

    save_path = forms.save_file(file_ext="json")
    if save_path:
        with open(save_path, "w") as f:
            json.dump(output, f, indent=2)
        script.get_output().print_md(
            "**PDA Export Complete**\n\n"
            "Exported {0} nodes, {1} members to:\n`{2}`\n\n"
            "Open this file in the PDA frame2d browser UI to add supports and loads.".format(
                n_nodes, n_members, save_path
            )
        )
```

**Key design decisions:**
- No f-strings: IronPython 2.7 compatibility. Use `.format()` everywhere.
- `FEET_TO_METRES = 0.3048`: explicit constant (Pitfall 4 — Revit XYZ is always in feet).
- `get_or_add_node`: deduplicates nodes within 1mm tolerance.
- Members array uses 1-based indices (PDA API convention).
- Default E=200GPa, I=1e-4, A=0.01 — generic steel values. User must update in UI.
- Canvas section has `origin: None` and empty arrays — no UI state from Revit.
- `forms.alert()` if no analytical members found (user-friendly error).
- `forms.save_file()` for native save dialog.
- `script.get_output().print_md()` for success message in pyRevit output window.
