## Component Inventory

All new components follow existing patterns in `ui/frame2d/style.css` and `ui/frame2d/script.js`. No new CSS classes are invented without reference to the existing pattern.

### 1. Per-Member Properties Panel (MODEL-01)

Triggered by: clicking a member in mode `memberProps` (new mode).

Interaction pattern: same as UDL (`mode === 'udl'` click-member path in script.js lines 120–130). On click: open an inline properties overlay rendered as a fixed-position panel beneath the sidebar, or use `prompt()` for each field in sequence (same as UDL/node loads). Recommended: use three sequential `prompt()` calls (E GPa, I cm⁴, A cm²) consistent with existing prompt-based interaction. No modal element needed.

If the user enters a value: store as `members[mi].E_override`, `members[mi].I_override`, `members[mi].A_override`. Leave as `null` for unset (falls back to global). Draw a 3px `#3f51b5` border around the member line on canvas to indicate an override is active.

Button label: `⊞ Per-Member E/I/A` — added to the "Member Properties" section in the sidebar.

### 2. Section Property Calculator Panel (MODEL-03)

Location: new `panel-section` in the sidebar, below "Material Properties". Always visible (permanent, not toggled).

Section heading: `Section Calculator` (11px uppercase `#888`, existing h3 style).

Contents:
- Select element: `<select>` with options `Rectangle`, `Circle`, `I-section`. Width 100%, font-size 12px.
- Rectangle inputs: `b (mm)` and `h (mm)` — two `<input type="number">` in the existing label/input pattern.
- Circle inputs: `d (mm)` — one input.
- I-section inputs: `b (mm)`, `H (mm)`, `tf (mm)`, `tw (mm)` — four inputs.
- "Calculate" button: styled as `.tool-btn` (not `.solve-btn`) — this is a utility action, not the primary CTA.
- Result display: two read-only lines below the button: `I = {value} cm⁴` and `A = {value} cm²`. Text color `#1565c0`. Font-size 12px. Hidden until Calculate is clicked.

The JS formulas live in the UI script (not a separate API call), duplicating the Python logic for simplicity at this scale (per CONTEXT.md D-09 discretion).

### 3. "Download Results" Link (MODEL-04)

Location: inside `.results-panel` `#resultsPanel`, above the `.results-grid`, rendered by `renderResults()` after a successful solve.

Element: `<a>` tag styled as an inline text link. Label: `Download results (JSON)`. Color: `#1a2744`. Underline. Hover: `#3f51b5`.

Behaviour: `href` set to an object URL (`URL.createObjectURL(blob)`) with `download` attribute set to `{solver}-results-{timestamp}.json`. Triggered programmatically or by user click. No separate button.

Appears in both `ui/frame2d/` and `ui/truss2d/` following identical pattern.

### 4. "Node Labels" Toggle Checkbox (MODEL-05)

Location: "Display" section in the frame2d sidebar, after the existing `chkSFD` checkbox row.

HTML pattern: identical to existing checkbox rows (`<label class="checkbox-label"><input type="checkbox" id="chkNodeLabels"> Node labels</label>`). Default: unchecked (overlay hidden).

Canvas render: after the main structure draw pass, if `chkNodeLabels.checked`, draw for each node: `N{i} [{base},{base+1},{base+2}]` where `i` is 0-based node index and `base = i*3+1`. Font: `600 11px Arial`. Fill: `#1a2744`. Offset: 8px to the right and 8px above the node circle centre. Does not block hit detection (drawn after interactions resolve).

### 5. Symbol Size Control (folded todo, D-16)

Location: "Display" section in the frame2d and truss2d sidebars, after the Node labels checkbox row.

Element: `<label>` with text `Symbol size ×` and a numeric input (`<input type="number" id="inputSymbolScale" value="1.0" min="0.5" max="2.0" step="0.1">`). Font-size 12px. Same label/input pattern as existing "Deflection scale ×".

Applies to: node circle radius, support triangle size, load arrow size — all multiplied by the scale value at draw time. Default 1.0. Range 0.5–2.0.

## Canvas Rendering Order

All new draw passes follow the established layering from CONTEXT.md code context:

1. Grid
2. Members (with per-member override indicator stroke)
3. Nodes
4. Supports
5. Loads
6. Deformed shape (if `chkDeflected` checked)
7. BMD (if `chkBMD` checked)
8. SFD (if `chkSFD` checked)
9. **Node label overlay** (NEW — if `chkNodeLabels` checked) — always last, never blocks hit detection
